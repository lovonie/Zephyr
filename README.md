Zephyr 使用和设置说明
适用范围：当前项目对应的已发布便携版 Zephyr
目标读者：Windows 10 / Windows 11 最终用户与进阶用户


一、文档说明


1. 本文档用于说明 Zephyr 的实际使用方法、设置方法、常见场景和特殊运行机制。
2. 本文档面向用户，不讲解源码、类名、测试名或发布脚本。
3. 本文档以当前程序界面与当前实现为准。若某一处界面文案与本文档存在轻微差异，应以程序实际显示为准。
4. Zephyr 是便携式程序。它强调：
   - 直接运行
   - 通过托盘图标操作
   - 主窗口和附属窗口联动
   - 大量行为可通过快捷设置和外观设置调整
   - 项目数据、配置、自启、资源替换均支持较高自由度



二、产品定位与运行方式


作用
Zephyr 是一个以托盘为中心、以主窗口为核心的桌面启动与管理工具。它支持：
- 管理项目、文件、文件夹和分组
- 通过主窗口快速启动目标
- 通过托盘菜单快速打开主窗口、快捷设置、外观设置和关于窗口
- 管理热键、自动关机、自启、图标、音效、序号显示等
- 导入导出系统配置、项目配置和项目包

操作方法
1. 直接运行 Zephyr.exe。
2. 程序启动后，主要通过托盘图标、主窗口、分组窗口、快捷设置窗口、外观设置窗口进行操作。

结果
程序启动后会进入常驻运行状态，托盘图标成为主要入口之一。

注意事项
1. 如果 Zephyr.exe 不在名为“Zephyr”的文件夹中，程序会自动创建 Zephyr 文件夹，并从该文件夹中的 Zephyr.exe 继续运行。
2. 新实例成功启动后，程序会尝试清理外层旧的 Zephyr.exe。
3. 如果程序目录下没有 config.json，首次运行时会自动生成。
4. 程序目录下会自动维护 Shortcuts 文件夹，用于手动放置快捷方式备份或手动导入项目。
5. 当系统分辨率或缩放率变化时，程序会自动重启，以确保窗口布局恢复到正确状态。



三、首次运行与首次推荐操作


作用
帮助用户理解 Zephyr 第一次启动时会自动发生什么，以及第一次建议完成哪些设置。

操作方法
1. 双击 Zephyr.exe。
2. 观察托盘区域是否出现 Zephyr 主图标。
3. 右键托盘图标，查看托盘菜单。
4. 打开“快捷”窗口与“外观”窗口，确认是否需要调整默认设置。

结果
首次运行时，通常会发生以下行为：
- 自动进入 Zephyr 文件夹运行
- 自动生成 config.json（如果缺失）
- 自动创建 Shortcuts 文件夹（如果缺失）
- 自动创建无后缀提示文件：请将需要添加的快捷方式放置到这里
- 如果系统中不存在有效自启项，程序可能会自动补建常规自启

注意事项
1. 首次运行后，请优先确认：
   - 主窗口位置是否符合你的习惯
   - 是否需要启用或调整热键
   - 是否需要启用或调整自启
   - 是否需要自定义托盘图标、大小写指示图标和音效
2. 若系统缩放或分辨率刚刚变化，程序可能会自动重启一次，这是正常行为。



四、托盘图标与托盘菜单


作用
托盘图标是 Zephyr 的常驻入口。通过主托盘图标和大小写指示图标，用户可以快速完成核心操作。

操作方法
1. 主托盘图标左键：
   - 打开或切换主窗口相关显示状态。
2. 主托盘图标右键：
   - 打开托盘菜单。
3. 大小写指示图标左键：
   - 切换 Caps Lock 状态。
4. 大小写指示图标右键：
   - 不打开托盘菜单。

结果
托盘菜单提供以下项目：
- 窗口
- 熄屏
- 外观
- 快捷
- 退出
- 关于

注意事项
1. 主托盘图标可设置为旋转或静止。
2. 大小写指示图标可设置为：
   - 大写
   - 小写
   - 隐藏
3. 若设置为“隐藏”，大小写指示图标不会显示在托盘中。
4. Windows 是否将托盘图标直接显示在任务栏通知区域，最终由系统和用户设置决定，程序不能强制将其固定显示在外层。
5. 当前版本会把主托盘图标和大小写指示图标作为两个独立的 Windows 通知区域条目进行记忆。
6. 如果你是从旧版本升级到当前版本，Windows 可能会把这两个图标视为新的通知区域条目，因此你可能需要在任务栏设置里分别重新设置一次“始终显示 / 放入溢出区”偏好。
7. 即使两个图标已经独立记忆，最终显示在外层还是溢出区、以及重启后的左右相对位置，仍由 Windows 通知区域和用户设置共同决定，程序本身不保证绝对固定位置。



五、主窗口


作用
主窗口用于展示和管理项目列表，是 Zephyr 的核心操作界面。

操作方法
1. 通过托盘左键或托盘菜单“窗口”打开主窗口。
2. 左键项目可启动目标。
3. 右键主窗口空白区或项目行，可打开右键菜单。
4. 通过拖拽操作，可调整顺序、分组和位置。

结果
主窗口支持以下主要能力：
- 展示项目列表
- 启动项目
- 添加文件、文件夹、分组或手动导入
- 项目更名、删除、固定
- 三种显示模式切换
- 排序与记忆滚动位置
- 锁定与置顶
- 分组项图标自动跟随组内内容变化
- 项目图标缺失时自动回退到当前程序托盘图标

注意事项
1. 主窗口默认高度不是固定像素高度，而是按“默认显示 25 个项目”或用户手动保存的项目数量来决定。
2. 主窗口默认宽度不是固定值，而是优先按当前最长项目的实际渲染宽度决定；只有测不到内容时才使用兜底宽度。
3. 用户一旦手动拖拽主窗口宽度或高度，程序会记住用户的手动结果。
4. 当主窗口项目很少甚至为空时，程序不会因为内容变少而自动把主窗口高度缩回内容总高度。
5. 主窗口中的分组项图标遵循以下规则：
   - 分组为空时，使用当前程序托盘图标。
   - 分组有项目时，使用分组内第一个项目的图标。
6. 如果某个项目无法提取图标，程序会使用当前程序托盘图标补位，不再留出空白图标位。



六、主窗口显示模式


作用
主窗口支持三种显示模式，以适配不同的空间与视觉需求。

操作方法
1. 在主窗口右键菜单中点击“模式”。
2. 在二级菜单中选择：
   - 图文
   - 文字
   - 图标

结果
1. 图文模式：
   同时显示图标和名称。
2. 文字模式：
   只显示名称，不显示图标。
3. 图标模式：
   只显示图标，不显示名称与序号列内容。

注意事项
1. 在窗口位于屏幕左侧时，从图文/文字切换到图标模式会向左收缩；从图标恢复到图文/文字模式会向右展开。
2. 在窗口位于屏幕右侧时，会保持右侧贴边方向进行收缩或展开。
3. 当主窗口为空时，切换到图标模式也不会缩成一条无法操作的细边，而会保留正常图标列宽度。



七、主窗口右键菜单


作用
主窗口右键菜单是添加、管理和窗口控制的主要入口。

操作方法
在主窗口空白区或项目上点击右键。

结果
主窗口右键菜单包含以下类型：

一类：添加
- 手动
- 文件
- 件夹
- 分组

二类：针对项目本身的操作（在选中具体项目时出现）
- 更名
- 删除
- 固定 / 释固
- 自启
  - 常规
  - 高级
  - 清除

三类：针对窗口本身的操作
- 模式
  - 图文
  - 文字
  - 图标
- 记忆 / 失忆
- 重排
- 置顶 / 撤顶
- 锁定 / 解锁

注意事项
1. “固定”后的项目不再用小锁标记，而是用右侧序号变红来表示已固定。
2. 右键菜单和右键菜单的子菜单，不会再被当作主窗口外部点击，因此即使菜单高度大于主窗口高度，也能正常点击超出部分。
3. “自启”二级菜单只会对真实可启动的项目显示；占位项、无效项或分组项不会显示这一组操作。
4. “自启”中的：
   - 常规：为当前项目创建普通权限的计划任务自启
   - 高级：为当前项目创建最高权限的计划任务自启
   - 清除：删除该项目对应的计划任务自启
5. 分组窗口右键菜单与主窗口右键菜单不同，但对具体项目也支持同样的“自启”二级菜单。
6. 右键二级菜单会以当前激活的一行作为对齐点，并优先向更适合容纳完整子菜单的一侧展开。



八、添加项目的 4 种方式


作用
Zephyr 支持从多个入口添加项目，以适应不同使用习惯。

操作方法
在主窗口右键菜单中打开“添加”。

结果
1. 手动
   - 直接打开 Zephyr.exe 所在目录下的 Shortcuts 文件夹。
   - 你可以把需要添加的快捷方式手动放进去。
2. 文件
   - 打开内置文件选择器。
   - 可直接选择文件，包括快捷方式文件。
3. 件夹
   - 打开内置文件选择器。
   - 选择文件夹后作为项目添加。
4. 分组
   - 新建一个分组项目。
- 默认名称为“新组”。

注意事项
1. 手动方式使用的是程序同目录的 Shortcuts 文件夹。
2. 如果 Shortcuts 文件夹或提示文件缺失，程序会自动补建。
3. “件夹”是界面当前文案，请按程序实际界面识别。
4. 新建分组默认名称已经固定为“新组”，不再出现乱码。



九、分组窗口


作用
分组窗口用于展示某个分组内部的项目列表。

操作方法
1. 在主窗口中点击分组项目。
2. 分组窗口会在主窗口附近打开。
3. 组内项目支持启动、拖拽、右键管理。

结果
分组窗口具备以下特征：
- 外观与主窗口尽量保持一致
- 可显示图标、文本、序号列
- 序号列也可独立隐藏或显示
- 右键菜单支持更名、删除、固定 / 释固
- 对真实可启动的组内项目，也支持“自启 -> 常规 / 高级 / 清除”

注意事项
1. 分组窗口默认宽度不再固定，而是优先按内容宽度决定；当组内内容为空时，其默认宽度与主窗口空内容时的默认宽度保持一致。
2. 分组窗口内容为空时，默认高度按 5 个项目高度计算。
3. 分组窗口不是简单贴着主窗口外壳展开，而是会尽量对齐到你点击的那个分组项所在行。
4. 分组窗口向上还是向下展开，不再只看主窗口整体位于屏幕上半区还是下半区，而是看你点击的分组项相对主窗口中心线的位置：
   - 分组项更靠上时，分组窗口优先向下展开
   - 分组项更靠下时，分组窗口优先向上展开
5. 若主窗口或分组窗口关闭或隐藏，部分附属窗口会自动跟随收起。
6. 当分组内第一个项目发生变化时，主窗口中的对应分组图标会实时刷新。



十、拖拽、排序、固定与分组


作用
帮助用户通过拖拽完成项目重排和分组整理。

操作方法
1. 在主窗口或分组窗口中按住项目拖动。
2. 观察拖拽高亮和插入指示线。
3. 松开鼠标完成排序或位置变更。

结果
1. 项目可以重新排序。
2. 项目可以进入分组或从分组中移出。
3. 固定项目会以红色序号显示，便于与普通项目区分。
4. 拖拽激活后，主窗口和分组窗口都会显示半透明的拖拽幽灵快照，并同时保留插入指示。

注意事项
1. 固定是布局语义的一部分，会影响排序与显示结果。
2. 程序支持拖拽时边缘自动滚动，但滚动速度和惯性会受外观设置影响。
3. 拖拽幽灵会跟手移动，结束拖拽、窗口隐藏或源行失效后会自动清空。
4. 主窗口中，未固定且未被手工拖拽排序锁定的项目，会参与“常用项目”自动排序：同一项目在最近一次重排之后点击达到 3 次及以上时，会优先显示。
5. 这种“常用项目”状态会随配置一起保存，因此正常退出或自动保存后，重启程序仍会保留主窗口中的提前结果。
6. 分组窗口不使用“点击 3 次优先”的规则；分组窗口内部仍然只按固定状态、组内手工顺序和基础名称排序来决定显示顺序。



十一、快捷设置窗口总览


作用
快捷设置窗口用于集中管理热键、自动关机、托盘图标、大小写指示图标、音效、序号列、配置、自启和日志。

操作方法
1. 通过托盘菜单“快捷”打开。
2. 快捷窗口与主窗口绑定显示。
3. 可手动调整宽度，程序会记住用户结果。

结果
快捷设置窗口包含以下项目：
- 快捷开启 / 快捷关闭
- 定时关机 / 计时关机
- 旋转 / 静止
- 大写 / 小写 / 隐藏
- 音效 / 音效 / 禁音
- 序号
- 配置
- 重置
- 日志
- 自启

注意事项
1. 快捷窗口首次无手动宽度时，默认宽度为 142。
2. 程序会自动检测系统真实自启状态，并同步到该窗口显示。
3. 某些按钮在特定模式下会被停用，例如“隐藏”时图标更换和默认不可点击，“禁音”时音效更换和默认不可点击。
4. 打开快捷窗口时，主窗口仍可继续正常使用，不会被锁定。
5. 若你在主窗口中点击分组项打开分组窗口，或直接点击已打开的分组窗口，快捷窗口会自动收起。



十二、热键


作用
通过全局热键快速唤出或收起主窗口，也可按序号直接打开项目或分组。

操作方法
1. 在快捷窗口的热键项中录入组合键。
2. 点击“保存”保存热键。
3. 点击“清除”清空热键。
4. 用左侧开关切换“快捷开启 / 快捷关闭”。
5. 只需要打开或关闭主窗口时，按下快捷键组合后直接松开即可。

结果
1. 热键保存后可长期生效。
2. 关闭热键时，已保存的组合键不会被删除，只是暂停响应。
3. 单独按下并松开快捷键组合本身时，会切换 Zephyr 主窗口；主窗口隐藏时打开并激活，主窗口已打开时关闭。
4. 按住快捷键组合不放时，可以继续输入项目序号。
5. 松开快捷键组合后，程序才会按已输入的数字一次性解析并执行。

注意事项
1. 默认组合键为 Ctrl + Shift + Z。
2. 清除会同步写入配置，不只是界面清空。
3. 输入规则如下：
   - 只要快捷键组合一直按着，输入数字之间没有时间限制。
   - 输入 2 位数字时，会按主窗口序号解析：
     - 对应普通项目时直接启动。
     - 对应分组项时打开该分组。
   - 输入 4 位数字时，前两位表示主窗口中的分组序号，后两位表示该分组内项目序号，松开快捷键后会直接启动该分组项目。
4. 输入非法位数，或输入了不存在的序号时，程序会直接忽略本轮操作，不弹提示。



十三、自动关机


作用
支持两种自动关机模式：
- 定时关机
- 计时关机

操作方法
1. 在快捷窗口中点击该项的切换按钮，在“定时关机”和“计时关机”之间切换。
2. 输入目标时间或倒计时时长。
3. 点击“确定”执行。
4. 点击“清除”取消。

结果
1. 两种模式互斥。
2. 切换到另一种模式时，原先模式已设置的自动关机会被自动清除。
3. 窗口下方会显示倒计时状态。

注意事项
1. 倒计时行现在只显示倒计时值，并居中显示。
2. 不论是定时关机还是计时关机，都会显示剩余时间。



十四、程序托盘主图标：旋转 / 静止


作用
控制主托盘图标是否旋转。

操作方法
1. 在快捷窗口中找到“旋转 / 静止”项。
2. 点击其图标按钮切换当前状态。
3. 可使用“更换 / 默认”替换或恢复主托盘图标资源。

结果
1. 旋转时，主托盘图标会按当前外观设置中的托盘动画参数运行。
2. 静止时，主托盘图标保持静态。

注意事项
1. 托盘动画不是简单匀速转动，而会受到“转速、风场、周期”等参数共同影响。
2. 点击切换后，状态会保存到配置中。



十五、大小写指示图标：大写 / 小写 / 隐藏


作用
控制 Caps Lock 指示图标的显示形态。

操作方法
1. 在快捷窗口中找到该项。
2. 点击图标按钮，在“大写 / 小写 / 隐藏”之间切换。
3. 若当前是“大写”或“小写”，可点击“更换 / 默认”替换或恢复对应图标。

结果
1. 大写：显示大写状态图标。
2. 小写：显示小写状态图标。
3. 隐藏：不显示大小写指示图标。

注意事项
1. 当切换到“隐藏”时，“更换 / 默认”会被停用。
2. 再切回“大写”或“小写”时，对应图标会重新显示。

二十一、自启


作用
控制 Zephyr 开机自启。

操作方法
1. 在快捷窗口中找到“自启”项。
2. 左侧开/关控制自启启用状态。
3. 右侧可选择：
   - 常规
   - 高级

结果
1. 常规：
   - 通过当前用户上下文的普通计划任务实现自启。
2. 高级：
   - 通过“最高权限”的计划任务实现自启。
3. 程序启动时会自动检测系统中的真实自启状态，并同步显示到快捷窗口。
4. 如果检测到旧路径遗留的高级自启，程序会自动重建为当前路径的常规自启，以保证下一次开机仍可正常启动。

注意事项
1. 当系统中没有有效自启项时，首次运行默认补建的是“常规自启”，不是“高级自启”。
2. 只有用户主动点击“高级”时，程序才会弹出管理员权限确认，并创建最高权限计划任务。
3. 被动状态刷新、启动同步、打开快捷窗口等操作，不会主动弹出管理员确认。
4. 如果 Zephyr.exe 路径发生变化，程序会检测旧的高级自启是否已经指向旧路径；检测到后，会自动降级重建为当前路径的常规自启，并清除旧高级自启。
5. 如果你仍然需要最高权限自启，请在新路径运行的 Zephyr 中再次主动点击“高级”，并确认管理员授权。
6. 关闭自启时，程序会同步删除计划任务和旧的启动目录遗留快捷方式。
7. 这里的“自启”是 Zephyr 程序本身的开机自启。
8. 如果你想让某个具体项目随系统启动，请在主窗口或分组窗口中对该项目点击右键，再使用“自启 -> 常规 / 高级 / 清除”。

二十三、外观窗口滑块的交互方式


作用
解释外观窗口中滑块的真实交互逻辑，避免误解。

操作方法
1. 直接点击某一项，可选中该项。
2. 在外观窗口中，带调色能力的项目点击后会打开独立调色窗口。
3. 点击滑轨本身时，只会选中项目，不会直接跳值。
4. 抓住 Thumb 后可立即拖动。
5. 项目激活后，也可以把鼠标放在滑轨范围内滚动滚轮微调。
6. 点击其他项会切换当前激活目标。

结果
1. 外观窗口和调色窗口都遵循相同的滑轨交互语义。
2. 调色窗口会复用同一个实例，不会每次点击都新建一个窗口。
3. 调色窗口默认宽度跟随当前外观窗口宽度，高度按内容自动适配。
4. 用户可以临时拉宽调色窗口，但这个宽度不会被持久保存。

注意事项
1. 鼠标滚轮只有在“当前激活项的滑轨宽度范围内”才调参数；在名称区、数字区或离开该项时，滚轮仍然滚页面。
2. 程序不会恢复 WPF 默认那种“点一下轨道就瞬间跳值”的行为。
3. 当点击靠近外观窗口下半区的项目时，调色窗口会优先向上展开，避免越出工作区底边。
4. 分组标题没有悬停效果，因为它们不是交互项。

二十五、手动添加与 Shortcuts 文件夹


作用
为用户提供一个可直接手工维护的快捷方式入口。

操作方法
1. 主窗口右键 -> 添加 -> 手动。
2. 打开的就是 Zephyr.exe 所在目录下的 Shortcuts 文件夹。
3. 将需要添加的快捷方式放入该文件夹。

结果
1. 程序会自动维护这个目录。
2. 目录中还会保留一个无后缀提示文件：请将需要添加的快捷方式放置到这里
3. 新进入导入链的快捷方式，程序会自动规范它们的显示名称；必要时也会同步规范 Shortcuts 目录中的真实快捷方式文件名。

注意事项
1. 若目录或提示文件缺失，程序会自动补建。
2. Shortcuts 同时也是项目包导入导出和快捷方式备份的重要目录之一。
3. 对于 `.lnk/.url` 快捷方式，程序会优先采用 Windows Shell 的本地化显示名称，再清理明确的噪音尾缀。
4. 当前会自动清理的典型尾缀包括：
   - `.exe`
   - `.exe - 快捷方式`
   - ` - 快捷方式`
5. `Portable` 不再属于自动清理尾缀；名称中带有 `Portable` 的快捷方式，程序会尽量原样保留。
6. 这类规范化主要作用于新导入项目，不会批量回写你已经绑定好的历史项目名称。
7. 对于控制面板、管理工具、运行等系统工具快捷方式，程序会尽量采用系统本地化名称和对应系统图标，而不是只按磁盘上的英文文件名或壳对象英文显示名显示。
8. 如果历史项目里保留了旧的自动识别名称，程序会在快捷方式刷新时尽量自动校正；但如果该名称已经被你手工改过，程序不会强行覆盖你的自定义名称。

二十七、特殊运行行为


作用
说明程序中那些不是直接由用户点击触发，但对用户体验有明显影响的自动行为。

操作方法
无须手动操作；以下行为由程序自动执行。

结果
1. 目录自举运行
   - 如果程序不在 Zephyr 文件夹中，会自动创建并迁移到该文件夹运行。
2. 首次生成配置
   - 若目录下没有 config.json，会自动生成。
3. 首次生成 Shortcuts 目录
   - 若目录下没有 Shortcuts，会自动创建。
4. 显示设置变化自动重启
   - 缩放率或分辨率变化时，程序自动重启以恢复布局正确性。
5. 自启状态自动检测与自修复
   - 启动时会检测系统真实自启状态。
   - 如果旧的高级自启仍指向旧路径，程序会自动降级重建当前路径的常规自启，保证下一次开机可正常启动。
6. 提示消息自动适配所属窗口
   - 普通提示消息会跟随主窗口字号显示。
   - 超过可用宽度时会自动换行。
   - 提示消息宽度不会超过所属窗口宽度。
   - 除关于窗口内容外，用户可见提示消息默认使用中文并补齐标点。
7. 快捷方式名称与图标自动规范化
   - 导入 `.lnk/.url` 时，程序会优先采用系统本地化显示名称。
   - 带有 `.exe`、`.exe - 快捷方式`、` - 快捷方式` 等明确尾缀的快捷方式，会被自动清理名称。
   - `Portable` 不再自动清理。
   - 系统工具快捷方式、DLL 资源图标、负资源 ID 图标等特殊图标会尽量自动解析。
8. 图标缺失自动补位
   - 分组为空时，主窗口中的分组图标使用当前程序托盘图标。
   - 分组有项目时，主窗口中的分组图标使用组内第一个项目的图标。
   - 普通项目提取不到图标时，也会回退到当前程序托盘图标。
9. 全局小手鼠标
   - Zephyr 所有窗口中的鼠标外观统一为小手样式。
   - 鼠标颜色、透明度和强度可在外观窗口中自定义。
   - 当鼠标进入窗口移动或调宽高的识别区时，小手会自动让出，不影响原本的拖拽和缩放操作。
10. 主窗口与图标预热
   - 托盘隐藏启动时，程序会优先保证托盘图标和全局输入尽快可用。
   - 主窗口内容与图标会在后台做必要预热，以改善首次打开和闲置后重开的响应。
11. Shortcuts 目录按需同步
   - 主窗口和分组窗口打开前，会根据 Shortcuts 的真实变化决定是否同步刷新。
   - 没有变化时，不会反复做不必要的全量扫描。

注意事项
1. 这些行为不是故障，而是程序设计的一部分。
2. 若用户不了解这些自动行为，最容易误解的是：
   - 程序为什么“自己重启”
   - 程序为什么“自己创建文件夹”
   - 为什么目录里会出现 config.json 和 Shortcuts
   - 为什么提示消息会在不同窗口中自动换行或跟随字号变化
   - 为什么旧高级自启会被自动变成常规自启

场景四：自定义大小写图标和音效
作用
让 Caps Lock 提示更符合个人习惯。

操作方法
1. 在快捷窗口中切到“大写 / 小写 / 隐藏”。
2. 分别替换 Caps On 和 Caps Off 图标。
3. 在音效项中切到“大”或“小”，替换对应音效。

结果
用户可拥有独立的大小写图标与开关音效。

注意事项
音效必须是小于 1 秒的 WAV。



三十、常见问题与排障


问题一：为什么我改了系统缩放，程序自己重启了？
答：
这是程序的设计行为。显示缩放或分辨率变化会影响窗口尺寸、贴边和绑定关系，程序会自动重启以恢复正确布局。

问题二：为什么程序目录里自己生成了 config.json？
答：
如果首次运行时没有 config.json，程序会自动生成，作为当前运行配置文件。

问题三：为什么会有 Shortcuts 文件夹？
答：
Shortcuts 是手动添加、快捷方式备份、项目包导入导出相关功能使用的目录，程序会自动维护它。

问题四：为什么自启状态和我印象中不一样？
答：
程序启动时会检测系统真实自启状态，并同步显示。当前默认自启是“常规计划任务”，而不是旧的启动目录快捷方式。如果程序发现旧路径遗留的“高级自启”已经失效，会自动把它改成当前路径的“常规自启”，以保证下一次开机仍可正常启动。

问题五：为什么“更换 / 默认”按钮有时点不了？
答：
1. 当大小写图标模式为“隐藏”时，图标更换 / 默认会停用。
2. 当音效模式为“禁”时，音效更换 / 默认会停用。

问题六：为什么托盘图标没有固定显示在任务栏外层？
答：
1. 这由 Windows 通知区域策略决定，程序本身不能强制把图标自动拖出溢出区。
2. 当前版本会把主图标和大小写图标作为两个独立通知区条目交给 Windows 记忆；如果你刚从旧版本升级，通常只需要在任务栏设置里分别重新设置一次显示偏好即可。

问题七：为什么程序运行后目录结构变了？
答：
如果 Zephyr.exe 不在 Zephyr 文件夹中，程序会自动创建 Zephyr 文件夹，并从该文件夹中的 Zephyr.exe 继续运行。

问题八：为什么我导入的快捷方式名称和文件名被改了？
答：
对于新进入导入链的 `.lnk/.url`，程序会优先采用系统本地化显示名称，并自动清理明确的噪音尾缀，例如 `Portable`、`.exe - 快捷方式`、` - 快捷方式`。如果你是直接把快捷方式放进 Shortcuts，程序在导入时还可能同步重命名 Shortcuts 中的真实快捷方式文件名，以保持显示名称、导入名称和实际备份文件名一致。

问题九：为什么有些项目或空分组显示成 Zephyr 的托盘图标？
答：
这是图标补位逻辑。空分组会显示当前程序托盘图标；分组有项目时会显示组内第一个项目的图标；普通项目提取不到图标时，也会用当前程序托盘图标代替缺位。



三十一、推荐使用顺序


1. 首次运行程序。
2. 检查托盘图标与主窗口是否正常。
3. 在主窗口中添加常用项目与分组。
4. 打开快捷窗口，确认：
   - 热键
   - 自启
   - 托盘图标
   - 大小写指示图标
   - 音效
5. 打开外观窗口，调整外观、布局和交互参数。
6. 用“配置 -> 出”导出系统配置和项目配置，形成自己的备份。

三十三、结束语


Zephyr 的设计目标不是“只提供一个启动面板”，而是提供一个可长期使用、可迁移、可自定义、可自修复、可通过托盘与快捷设置持续调整的便携式桌面工作台。

如果你是第一次使用，建议先从以下三件事开始：
1. 建立自己的项目与分组结构
2. 设置自启与热键
3. 备份系统配置和项目包

完成这三步后，Zephyr 基本就能进入稳定的长期使用状态。
